# Работа с версткой

## Получение верстки
* Верстка лежит в виде исходников в папке mockup, как git модуль;
* После клонирования репозитория https://gitlab.com/web-aim.ru/mirvendinga.ru;
* Написать `git module init`;
* Написать `git module update`, после чего верстка должна склонироваться в папку `/local/templates/mirvendinga/mockup/`;
* Чтобы верстка правильно применялась, ее нужно собрать. Сборщик работает на `node 12.14.1`;
* Для установки node_modules нужно написать `npm ci` - `ci` вместо `i` используется для того, чтобы не изменять `package-lock.json`;
* Написать `npm run zip` для сборки папки `dist`, в которой находятся стили, скрипты верстки, * иконки;

## Обновление верстки
* Перейти в папку `/local/templates/mirvendinga/mockup/`;
* Написать `git pull`;
* После каждого обновления верстки ее нужно пересобирать командой `npm run zip`.
